pod的基本用法

pod是k8s调度的最小单位，pod会根据rc中定义的replicas副本数生成pod个数，一般情况下一个pod中只存有一个镜像，但是在某些场景下一个pod需要存在多个镜像，只需要将两个docker都绑定在一起即可，如下是一个编排以下的场景是启动了一个java程序两个前端同时要访问这一个后端

静态pod

静态pod是kubele进行管理并仅存特定node，不能通过apiserver管理因为具体也没有实践过就不深究了后面研究好了再补上

pod的配置管理

configmap

configmap使用场景

  • 生成容器环境变量
  • 设置容器启动参数命令
  • 以volume的形式挂在为容器内部文件或目录
    configmap都是以k:v形式存储的 可以直接通过yaml或者命令生成

创建configmap

如下是最简单的configmap的创建使用kubectl create -f cm-appvars.yaml然后查看kubectl get cm查看是否创建成功
使用kubectl describe cm cm-appvars查看详细信息

下面这个例子描述两个配置文件server.xml和logging.properties定义的用法设置的key是别名，values是文本的全部内容

同样的既然可以使用yaml也可以使用命令进行生产configmap并且可添加多个参数

  • 如下当前目录如果有server.xml就可以创建包含这个文件的configmapkubectl create configmap cm-server.xml --from-file=server.xml
  • 如果目录有两个配置文件kubectl create configmap cm-appconf --from-file=configfiles
  • –from-literal 同样可以创建但是未实践后面补上
